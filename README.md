Firebase Integration

This project uses Firebase as the backend infrastructure to provide real-time data synchronization, authentication, cloud storage, and scalable hosting for the Service Order and Technical Management System.

Technologies Used
Firebase Authentication
Cloud Firestore
Firebase Storage
Firebase Hosting
Firebase Realtime Database (optional)
JavaScript / HTML / CSS Frontend
Android Integration (future support)
Main Firebase Features
User Authentication

Firebase Authentication is used to securely manage user access.

Supported Features
User login and logout
Protected access to system data
Session persistence
Multi-user support
import { getAuth, signInWithEmailAndPassword } from "firebase/auth";

const auth = getAuth();

signInWithEmailAndPassword(auth, email, password)
Cloud Firestore Database

Firestore stores all system information in the cloud.

Stored Data
Service Orders
Client information
Technical reports
Equipment details
Service status
Dates and timestamps
Users
Technician accounts
Permissions
Activity records
Reports
Inspection data
Maintenance history
Generated PDF references
Firebase Storage

Firebase Storage is used to upload and manage files generated by the system.

Stored Files
Technical photos
Inspection images
Generated PDFs
Attachments

Example upload:

import { getStorage, ref, uploadBytes } from "firebase/storage";

const storage = getStorage();
const storageRef = ref(storage, 'reports/report1.pdf');

uploadBytes(storageRef, file);
Real-Time Synchronization

The system uses Firebase real-time capabilities to synchronize information instantly between devices.

Benefits
Instant updates
Cloud backup
Multi-device access
Online management from PC or mobile
Hosting

The application can be deployed directly using Firebase Hosting.

Deployment
firebase deploy
Security Rules

Firebase Security Rules are implemented to protect sensitive data and ensure only authenticated users can access the system.

Example:

rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {
    match /serviceOrders/{document} {
      allow read, write: if request.auth != null;
    }
  }
}
